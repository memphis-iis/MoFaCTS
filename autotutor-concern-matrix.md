## Production Trace Takeaways

1. Confidence interval: the authored M2 repair question asks whether the interval estimates individual scores or the population mean. The learner answered "The population mean" multiple times, but M2 remained current and progress stayed at 0. This suggests a repair-resolution failure at the scorer/planner boundary, possibly made worse by an authored repair question that is too narrow to satisfy the broader repeated-sampling expectations.
2. NVC: the learner covered E1, E2, and E3 substantially, reached about 0.41 displayed progress, and still had no completion after 15 turns. The content burden is real, but the first prompt problem is better understood as too little conversational invitation: it sounded like the learner needed to know the target answer rather than simply start talking.
3. NVC optional E7 is included in the progress denominator even though it is optional. That is a concrete content-software mismatch.
4. Both traces show no `autotutor-complete` action. That does not prove the design is broken, but it does strengthen concerns about staging, bottom-out, and completion policy.

## Highest Priority Corrections

1. Guarantee bottom-out synthesis and separate mastery from max-turn/session-ended state.
2. Add a first-class learner contribution classifier for `idk`, help-seeking, uncertainty, affect, meta-comments, questions, off-task turns, and assertions.
3. Make the active target expectation more transparent in the dialogue.
4. Fix progress/completion to use required expectations by default, especially for NVC where E7 is optional.
5. Add calibration tests for semantic scoring against expert-labeled sample responses, including misconception repair turns.
6. Revise the stimulus files where content is the likely cause: the NVC opening prompt has been softened into a discussion starter; next add explicit repair criteria for misconceptions and multiple repair-question variants where production showed looping.

## Implementation Notes

- The fixes should preserve the current no-silent-fallback rule: if the scorer omits fields, invents IDs, or violates the selected plan, the system should fail clearly.
- Most proposed options belong under `autotutorsession` because they affect AutoTutor runtime policy, not ordinary MoFaCTS card behavior.
- The non-negotiable invariants should not be optional: max-turn termination is not mastery, learner-generated coverage must remain distinct from tutor assertions, and malformed model output must not be silently accepted.
