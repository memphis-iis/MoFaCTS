Phase 6.2 plan:
- The primary goal of Phase 6.2 is to keep `serverComposition.ts` honest as a small server composition/registration entrypoint, not let business logic drift back into it.
- Helper extraction by itself is not enough. To reduce file size, ownership of whole responsibility clusters must move out of the composition entrypoint.
- The target end state is for `serverComposition.ts` to be mostly dependency wiring, method registration, runtime registration, and startup invocation, with business logic and cluster-local helpers living in dedicated modules.
- The intended module shape should be explicit. Representative target groups are:
  - `server/methods/authMethods.ts`
  - `server/methods/packageMethods.ts`
  - `server/methods/themeMethods.ts`
  - `server/methods/adminMethods.ts`
  - `server/methods/analyticsMethods.ts`
  - `server/methods/contentMethods.ts`
- These filenames are planning targets, not a rigid one-shot rewrite requirement, but the remaining work should move toward this kind of grouped method ownership rather than leaving the responsibilities inside `serverComposition.ts`.
- `A39` and `A42` therefore should be treated as the real file-size-reduction strategy, not as vague umbrella rows to defer indefinitely.
