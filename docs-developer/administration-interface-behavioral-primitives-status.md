# Administration Interface Behavioral Primitives Status

## Purpose

This file records implementation evidence for
`administration-interface-behavioral-primitives-plan.md`. A primitive is listed
as implemented only after it has tests and a real management-page consumer.

## Current Slice

Phases 0 through 5D are source-migrated. Phase 6 table behavior is migrated
through the identified management tables. Phase 7 form semantics is advanced
across the named management and authoring surfaces. Theme specialization,
Practice-shell stability, confirmation-controller consolidation, and CSS
ownership cleanup are now source-migrated in focused Phase 8/9 slices.

The remaining completion work is the plan-required runtime and accessibility
verification. Runtime verification remains intentionally deferred for the
current no-verification touch-up sequence.

## Phase 9 Deletion Search Classification

- `requestInlineConfirmation`: removed from the migrated APKG and Content
  Manager surfaces; their domain-named request bridges delegate to the shared
  controller.
- `sortable-useradmin`, `onclick="navigateToStudentReporting`,
  `window.themeSaveTimeout`, and `window.themeColorSaveTimeout`: no current
  client-source hits; removed from the migrated surfaces.
- `customLoading`: intentionally retained as an empty route contract, as
  recorded above; it does not render a spinner or transient message.
- `page-header-title` / `page-header-text`: the remaining heading-class hit is
  the non-management `experimentError` route. Remaining CSS hits are theme
  color variable names, not duplicate-heading selectors.
- `requestAnimationFrame`: remaining hits are learner practice paint timing,
  sign-in visibility timing, and the Home return-tour animation; they are not
  management-page initialization paths.
- `alert(` / `confirm(`: remaining hits are learner, router, and error-report
  flows outside the migrated management surfaces. No native `confirm()` hit
  remains in the client source.
